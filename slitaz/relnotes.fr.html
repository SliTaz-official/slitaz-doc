Présentation

Bienvenue SliTaz 3.0! Petite, rapide, simple à utiliser et personnaliser, la nouvelle version de stable de SliTaz paraît après un an de développement.

  • Scripts de démarrage réécrits et plus rapide
  • Amélioration des outils maison
  • Better theme and integration
  • Adoption de Midori comme navigateur par défaut
  • Meilleurs support du matériel. En particulier les cartes wiFi
  • Virtualisation rapide avec lguest
  • SliTaz pro: OpenERP, LAMP, Glpi
  • More contributors and stronger community
  • Simple à personnaliser
  • Les paquets sont compilés automatiquement par un robot (le build bot Tazbb)
  • 900 paquets ajouté et beaucoup mis à jour
  • Plus de 2800 commits réalisés par les contributeurs

L'assistance aux utilisateurs est assurés à travers une liste de diffusion et le forum officiel: forum.slitaz.org

Matériel supporté

SliTaz GNU/Linux supporte les machines à base de processeurs Intel i486 ou x86 . Un minimum de 192Mo de mémoire sont nécessaire pour utiliser le core LiveCD. 64Mo pour la saveur "slitaz-loram" flavor et 16Mo pour la saveur "slitaz-loram-cdrom".

Avec la saveur slitaz-loram, le système est moins réactif, mais permet une installation de SliTaz en mode graphique sur de très vièlles machines. Une fois installé, SliTaz fontionne avec au minimum 16Mo de mémoire. Mais oublié Firefox pour surfer sur Internet ! Préférez un navigateur en mode text, telque 'links' par exemple.

SliTaz 3.0 contient tous les pilotes et outils, comme 915resolution, pour fonctionner sur une large gamme de netbooks (Eeepc, HP mini, Aspire One etc.).

LiveCD à Saveurs

SliTaz GNU/Linux est distribuée sous la forme d'un LiveCD amorçable SliTaz GNU/Linux is distributed as a bootable LiveCD allowing you to use it as a nomad system or graphically install to the hard drive and retain the use of your previous system including all settings, applications, documents, etc.

The LiveCD can also be customised and easily rebuilt both graphically or from the command line. Install your own custom set of packages, or simply use one of the preset flavors on the mirror. Then simply generate your distribution with the "Tazlito" tool.

Kernel and toolchain

SliTaz 3.0 comes with a new but very well tested Kernel and a complete new toolchain from witch the all system is build. The Linux kernel version is 2.6.30.6 who is split in separate package for additionnal driver or features. This time we also built-in some more drivers to have a faster kernel boot.

The new toolchain comprise: Binutils 2.19.1, GCC 4.4.1, Glibc 2.10.1

HD install and LiveUSB

The installation is fully automated and can be done graphically or in text mode. In this version the installer will let you use a separate partition for /home, configure root password and chose default user login. Installer now also support intallation from a LiveUSB session.

Full USB install are also supported and some time needs a rootdelay=10 as a kernel argument (in GRUB menu.lst).

Software Packages

SliTaz 3.0 have 2291 package in the database, a wide variety of packages have been commited and many force went to professionnal grade software such as OpenERP, MySQL, Glpi.

On the update side, mainly all the packages have been update, including the kernel, all the toolchain, Xorg (7.4), GTK and QT. Package format have also changed and now use lzma for better compression. Packages are also checked by Tazwok to ensure FSH is followed and built automaticaly by SliTaz Build Bot: bb.slitaz.org

Core Desktop

The core desktop provide a selection of multi-use packages for surfing the web, listening to music, audio editing, image manipulation, developing or burning to optical media. It's just one click in the application menu to find software installed by category.

A lot of attention have been ported the the core desktop, all applications are well integrate and a special SliTaz theme (icons, Openbox, GTK) have been done.

From 1.0 to 2.0

The SliTaz GNU/linux installer offers an update function allowing you to upgrade from a '2.0' to '3.0' version. To upgrade the system you first need to boot the 'Stable' LiveCD, launch the installer, select upgrade and then specify the partition containing the system that you want to update. The installer will then clean out the system and reinstall all the packages not present on the CD from the mirror. When this has finished you can reboot with your new version of SliTaz.

Note that the installer will keep a copy of the list of packages and a complete archive of the /etc directory (etc.tar.gz) in /var/lib/slitaz-install.

To upgrade a 2.0 to 3.0 it is also possible to use the package manager "Tazpkg" via the 'set-release' function, but beware this is not yet proven and may require some manual intervention. If you choose this way you must first install 'tazpkg' from 3.0, because package format have change.
