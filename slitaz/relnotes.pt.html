Introdução

Bem-vindo ao SliTaz 3.0! Uma distribuição simples, rápida, customizável, poderosa e inacreditavelmente pequena! A nova versão estável do SliTaz é lançada após um ano de desenvolvimento.

  • Os scripts de boot foram reescritos e suportam boot ultrarrapido
  • Novas funções incluídas em todas as ferramentas gráficas do sistema (tazctrlbox, tazhw, wifibox, burnbox, mountbox, netbox, serverbox, instalador do SliTaz)
  • Mudança de servidor gráfico do Xvesa para Xorg
  • Mudança para o Midori como navegador web padrão
  • Melhor suporte a hardware - placas de rede, webcams, 3G-modems, impressoras, scanners
  • Melhor tema e integração de ambiente desktop
  • Melhor virtualização com o lguest
  • SliTaz pro: OpenERP, LAMP, GLPI e mais
  • Customização mais fácil do liveCD e liveUSB com o tazlitobox e o tazusbbox
  • O Tazpkg suporta novas funções, incluindo a conversão de pacotes deb/rpm/arch/slackware/ipk para o formato nativo do SliTaz
  • Os pacotes são automaticamente compilados por um robô de compilação (Tazbb)
  • Mais contribuidores e comunidade mais forte
  • 900 pacotes adicionados e várias atualizações
  • Mais de 2800 commits feitos pelos contribuidores

Suporte técnico é fornecido aos usuários da distribuição por meio da lista de discussão e pelo fórum oficial: forum.slitaz.org

Hardware Suportado

O SliTaz GNU/Linux suporta todas as máquinas baseadas nos processadores compatíveis Intel i486 ou x86. A memória mínima recomendável é 192MB para utilização do LiveCD padrão. Para a variante "slitaz-loram" é recomendável 64MB e para a "slitaz-loram-cdrom" recomendamos 16MB de memória RAM.

O SliTaz 3.0 fornece todos os drivers e ferramentas necessárias, tais como 915resolution, para suporte a vários tipos de netbooks (Eeepc, HP mini, Acer Aspire One, etc).

A variante slitaz-loram permite a instalação gráfica do SliTaz em máquinas antigas com recursos limitados. O LiveCD pode ser pouco responsivo mas, quando instalado, funciona bem com o mínimo de 16MB de memória. Em máquinas muito antigas, recomendamos a utilização de um navegador web em modo texto, como o 'links'.

Variantes do LiveCD

O SliTaz GNU/Linux é distribuido como um LiveCD bootável. Você pode usar o LiveCD como um sistema normal ou utilizá-lo para instalar o SliTaz no disco rígido. A instalação ou atualizar uma instalação do SliTaz em disco rígido faz com que você mantenha suas modificações, configurações e documentos.

O LiveCD também pode ser customizado facilmente tanto por uma interface gráfica quanto pela linha de comando. Você pode instalar seus pacotes preferidos ou utilizar uma das variantes existentes no mirror. Então, gere sua distribuição personalizada utilizando a ferramenta 'Tazlito'.

A imagem ISO agora é híbrida: ela pode ser copiada em uma mídia USB sem a necessidade de formatação, utilizando-se o comando dd.

Kernel e toolchain

O SliTaz 3.0 fornece um novo porém bem testado kernel linux 2.6.30.6 e um toolchain (ferramentas básicas de compilação) completamente novo, no qual o sistema foi construído inteiramente. Este kernel está separado em vários pacotes, que suportam vários drivers e aspectos do sistema. Até agora, adicionamos alguns drivers que facilitam um boot mais rápido do kernel.

O novo toolchain consiste de: Binutils 2.19.1, GCC 4.4.1 e Glibc 2.10.1

Suporte UTF-8

Com este lançamento os usuários do SliTaz serão beneficiados com o suporte a UTF-8. Agora podemos utilizar arquivos PO para traduzir todas as ferramentas e utilitários, gráficas ou de linha de comando. Com um suporte melhor à internacionalização (russo, português, chinês e mais) os contribuidores do projeto podem usar ferramentas gráficas como o poedit para traduções seguindo os padrões do GNU gettext. O labs possui um projeto i18n que permite o upload de arquivos que podem ser incluídos na distribuição. Os pacotes de localização possuem mais mensagens de aplicativos traduzidas para o sistema base e para alguns pacotes extra, como o Gimp, Abiword, Gnumeric, etc.

Servidor Xorg

O SliTaz agora utiliza o Xorg como servidor gráfico. Com o utilizário tazx pode-se reconfigurar o servidor e selecionar o driver correto para determinada placa de vídeo. Por padrão o Xorg fornece o driver vesa que é suficiente para funcionar na maioria das placas. Se algum driver for instalado, é possível que haja a necessidade de configurar manualmente o xorg.conf. Esperamos que nossa crescente comunidade de contribuidores possa oferecer e fornecer suporte a uma variante oficial com o antes utilizado Xvesa.

Instalação no HD e LiveUSB

O processo de instalação é totalmente automatizado e pode ser feito graficamente ou em modo texto. O instalador do SliTaz agora permite a utilização de uma partição /home separada, configuração da senha de root e de usuário padrão. Também suporta a instalação a partir de uma sessão LiveUSB.

A instalação em mídias USB é suportada, sendo que em alguns casos é necessária a utilização do argumento de kernel rootdelay=10, adicionado ao arquivo menu.lst (GRUB). Isto assegura que o kernel detecte a mídia antes de montá-la em / (root).

Pacotes

O SliTaz 3.0 possui uma grande variedade de 2.291 pacotes adicionados ao repositório. O Gerenciador de Pacotes tazpkg agora suporta a conversão de pacotes deb/rpm/arch/slackware/ipk para o formato padrão tazpkg. Também mantemos vários pacotes para utilização profissional ou empresas, tais como OpenERP, MySQL e GLPI.

Os pacotes estão, em sua maioria, atualizados, incluindo o kernel, o toolchain, o Xorg (7.4), GTK e QT. O formato de pacotes mudou, suportando o lzma para melhor compressão, o que permite downloads mais rápidos de pacotes. O tazwok checa os pacotes para assegurar que seu formato respeita os padrões FHS (Filesystem Hierarchy Standard) e o robô de compilação (bb.slitaz.org) compila-os automaticamente.

Desktop Básico

Muita atenção foi dada ao desktop básico. Todos os aplicativos estão bem integrados e um tema foi especialmente criado (ícones, Openbox, GTK).

O desktop básico fornece uma seleção de pacotes para navegar na web, ouvir música, editar áudio, manipular imagens, desenvolvimento e gravação de mídias. O menu permite encontrar os pacotes divididos por categorias.

Segurança

O SliTaz está mais seguro e robusto. As regras de Firewall foram atualizadas, sendo que este pode ser instalado por meio da ferramenta gráfica tazhw e gerenciado pela serverbox. A ferramenta mountbox, que possui como dependências os pacotes dmsetup, linux-md e devmapper, gerencia dispositivos crypto com o devmapper. As ferramentas de configuração Crypt permitem a encriptação de dispositivos de bloco.

Da versão 2.0 para a 3.0

O instalador do SliTaz oferece uma função que permite a atualização da versão estável 2.0 para a versão estável 3.0. Para atualizar o sistema deve-se inicializar o sistema com o LiveCD estável, executar o instalador, selecionar a opção "upgrade" (atualizar) e especificar a partição na qual o sistema a ser atualizado se encontra. O instalador irá, então, limpar o sistema e reinstalar todos os pacotes não presentes no CD a partir do mirror. Quando o processo estiver terminado, reinicie o sistema com sua nova versão instalada do SliTaz.

O instalador irá manter uma cópia da lista de pacotes e um arquivo completo do diretório /etc (etc.tar.gz) gravados em /var/lib/slitaz-install.

A atualização da versão 2.0 para a 3.0 também é possível através do gerenciador de pacotes tazpkg, através da função 'set-release', porém avisamos que esta forma ainda não está totalmente testada e pode requerer alguma intervenção manual. Caso se escolha este método, deve-se instalar primeiramente o tazpkg da versão 3.0, devido à mudança no formato dos pacotes.
